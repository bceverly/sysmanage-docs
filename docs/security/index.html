Agent Approval System

SysManage implements a manual approval system for agent registration to ensure only authorized hosts can connect:

Approval Workflow

  1. Agent Registration: New agents register via `/host/register` endpoint with "pending" status
  2. Connection Blocked: Agents with "pending" or "rejected" status cannot establish WebSocket connections
  3. Administrator Approval: Administrators review and approve agents through the web interface
  4. Certificate Generation: Server generates unique client certificates for approved hosts
  5. Secure Connection: Approved agents can establish full WebSocket connectivity with mTLS

Security Benefits

  • Authorized Access Only: Prevents unauthorized agents from connecting
  • Manual Review: Administrators can verify each host before approval
  • Audit Trail: All approvals and rejections are logged
  • Certificate-Based Security: Approved hosts receive unique certificates for enhanced security

Vulnerability Reporting

We take security seriously and appreciate responsible disclosure of security vulnerabilities.

üîí For Security Issues (DO NOT create public issues)

  1. Email: Send details to security@sysmanage.org
  2. Include:
    • Description of the vulnerability
    • Steps to reproduce
    • Potential impact assessment
    • Suggested fix (if available)
  3. Response Time: We will acknowledge receipt within 48 hours
  4. Updates: You'll receive updates every 7 days until resolution

Responsible Disclosure Process

  1. Initial Report: Security researcher reports issue privately
  2. Acknowledgment: We confirm receipt and begin investigation
  3. Investigation: We assess impact and develop fixes
  4. Fix Development: Patches are developed and tested
  5. Coordinated Release: Fix is released with security advisory
  6. Public Disclosure: Details shared after fix is available
