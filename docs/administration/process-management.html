Process Management

View the running processes on any managed host and terminate them directly from the SysManage web UI, without opening an SSH session.

Overview

Each SysManage agent periodically collects a snapshot of the processes running on its host and reports it to the server. The snapshot appears on the host's Processes tab, sorted by resource usage, so you can spot a runaway process at a glance and stop it in one click.

  • Cross-platform: collected with psutil, so the same data is available on Linux, macOS, the BSDs, and Windows.
  • Resource-ranked: processes are sorted by CPU and memory so the heaviest are at the top.
  • Actionable: terminate a process (SIGTERM) or force-kill it (SIGKILL) from the UI, subject to a permission check.
  • On-demand refresh: request a fresh snapshot at any time instead of waiting for the next periodic cycle.

What is collected

For each process the agent reports the following fields:

  • PID / Parent PID: the process and parent process identifiers.
  • Name: the executable name.
  • User: the account the process runs as.
  • CPU %: whole-machine CPU usage (normalised across cores).
  • Memory % / RSS: memory usage as a percentage and as resident set size.
  • Status: running, sleeping, zombie, and so on.
  • Command line: the full command that launched the process.
  • Started at: when the process began.

To keep reports small on very busy hosts, the snapshot is capped at the most resource-hungry processes; when a cap is applied it is noted so nothing is dropped silently.

Viewing processes

  1. Open a host from the Hosts page.
  2. Select the Processes tab.
  3. Use the search box to filter by process name, user, or command line, and click a column header to re-sort.
  4. Click Refresh to ask the agent for a current snapshot; the list updates shortly after the agent responds.

The timestamp of the most recent snapshot is shown above the table, converted to your browser's local time zone.

Terminating a process

Users with the Kill Host Process security role see a Kill action on each row. Terminating a process:

  • Sends SIGTERM by default (a graceful stop), or SIGKILL if you tick Force kill.
  • Requires the agent to be running in privileged mode; otherwise the action is disabled.
  • Includes a safety check: if the process at that PID has been recycled to a different program since the snapshot, the kill is refused.
  • Is recorded in the audit log, and the process list refreshes automatically afterwards.

Grant the Kill Host Process role from User Management to operators who should be allowed to terminate processes. Viewing processes only requires the View Host Details role.

Process-based alerts (Professional+)

With the Professional+ alerting engine you can create a process_resource alert rule that fires when any process on a host exceeds a CPU and/or memory threshold — optionally filtered to a specific process name. See Monitoring & Alerts for how to define alert rules and notification channels.
