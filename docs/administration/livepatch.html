Livepatch Integration (Ubuntu)

See the Canonical Livepatch kernel live-patching status of your Ubuntu hosts — patched kernel, patch state, and applied CVE fixes — right in the host's Ubuntu Pro tab.

Overview

Canonical Livepatch applies critical kernel security fixes without rebooting. It is delivered as part of Ubuntu Pro, so SysManage's existing Ubuntu Pro support already lets you enable or disable the livepatch service. This feature adds the livepatch-specific detail that the generic service view doesn't show, so you can confirm at a glance that a host's kernel is fully patched.

When the livepatch service is enabled on a host, the agent runs canonical-livepatch status and reports the result to the server. If the client is not installed, the host simply shows the standard service toggle and no detail card.

What is shown

The Kernel Livepatch card on the host's Ubuntu Pro tab shows:

  • Patch status: for example applied, nothing-to-apply, or a state needing attention.
  • Patched kernel: the running kernel that livepatch is protecting.
  • Patch version: the livepatch payload version currently applied.
  • Client version: the canonical-livepatch client version.
  • Last check-in: when the client last contacted Canonical (shown in your local time zone).
  • Applied fixes: the CVEs addressed by the current livepatch.

Enabling Livepatch

Livepatch is an Ubuntu Pro service. On a host that is attached to Ubuntu Pro, open the host's Ubuntu Pro tab and enable the livepatch service — the same way you enable any other Pro service. Once it is running and the agent reports a fresh status, the Kernel Livepatch detail card appears. To attach a host to Ubuntu Pro in the first place, see the Ubuntu Pro settings in the Settings Guide.

The agent must be running in privileged mode to change Pro services. Viewing the livepatch status requires no special role beyond access to host details.

Notes

  • Livepatch is Ubuntu-specific; the card only appears on Ubuntu hosts with the service enabled.
  • Status is refreshed as part of the agent's normal reporting cycle, so a newly-applied patch appears within a few minutes.
  • Livepatch covers the running kernel between reboots; a kernel upgrade still requires a reboot to take effect, which SysManage's reboot tooling can coordinate.
