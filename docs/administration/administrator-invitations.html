Administrator Invitations

Invite people to SysManage by email with their security roles pre-assigned. The recipient accepts through a one-time link that creates their account and sets their password.

Overview

Instead of creating an account and separately communicating credentials, an administrator can send an invitation. SysManage emails the recipient a secure link; when they follow it they choose their own password and their account is created with the roles you selected. This keeps password creation in the user's hands and gives you role assignment up front.

  • Role-aware: pick the security roles (and optional administrator flag) the new account should have.
  • Time-limited: invitations expire after 7 days.
  • Single-use: the token is consumed when the account is created and cannot be reused.
  • Manageable: pending invitations can be re-sent or revoked at any time.

Prerequisites

  • Email must be configured — invitations are delivered by email, so the server's email settings must be working. See the Settings Guide.
  • Add User role — you need the Add User security role to create or manage invitations.

Sending an invitation

  1. Open the Users page and click Invitations.
  2. Enter the recipient's email address and, optionally, their first and last name.
  3. Select the security roles to grant, and tick Grant administrator access if the account should be an administrator.
  4. Click Send Invitation. The recipient receives an email with an Accept Invitation link.

If an account already exists for that address, or the invitee is invited again, the previous pending invitation is superseded so there is only ever one live link per address.

Accepting an invitation

The recipient clicks the link in the email, which opens the Accept Invitation page. They confirm their name, choose a password, and submit. Their account is created immediately with the assigned roles, and they can log in right away. Invalid or expired links show a clear message asking them to request a new invitation.

Managing pending invitations

The Invitations dialog lists every invitation with its status — Pending, Accepted, Revoked, or Expired. For a pending invitation you can:

  • Resend — email the invitation link again (for example if the first message was missed).
  • Revoke — invalidate the link so it can no longer be used.
