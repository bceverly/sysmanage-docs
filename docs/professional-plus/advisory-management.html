Advisory Management

Patch by advisory rather than by raw package. The advisory engine ingests vendor security notices, maps each advisory to the CVEs and packages it fixes, and computes exactly which advisories apply to each host — so you can remediate a named advisory across the fleet in one action.

Overview

SysManage already tracks CVEs and installed packages. Advisory Management adds the vendor-advisory layer on top: an advisory (for example Ubuntu USN-6700-1 or Red Hat RHSA-2024:1234) bundles a set of CVEs and the fixed package versions that resolve them for a given OS release. The engine keeps a global advisory catalog and, per host, works out which advisories are applicable — i.e. the host has an affected package installed at a version below the advisory's fix.

Supported advisory sources: Ubuntu USN, Red Hat RHSA / RHBA / RHEA, SUSE-SU / openSUSE-SU, Debian DSA, and FreeBSD-SA.

The advisory catalog is shared

Vendor advisory data is identical for every customer, so the catalog is stored once and shared across all tenants — the same model SysManage uses for its CVE database. Only the per-host applicability results are tenant-scoped. Ingestion runs server-wide on a schedule; you never re-download advisories per tenant.

Applicable advisories per host

Open a host's detail page and select the Advisories tab to see every advisory that currently applies to it, grouped by advisory with the affected packages and their installed → fixed versions.

  • Recompute re-evaluates the host against the current catalog (for example after an ingestion refresh or a package change).
  • Install queues installation of every package the advisory fixes, using SysManage's normal package-install path — so it is audited and honours any maintenance window configured for the host.

Fleet dashboard

The Advisories page in the main navigation summarises applicable advisories across the fleet — total applicable, hosts affected, and a breakdown by severity — above a catalog of advisories you can filter by severity, type (Security / Bugfix / Enhancement), and source, or search by advisory ID or title.

Advisory ↔ CVE mapping

Each advisory links to the CVEs it addresses, so you can pivot between the CVE-centric Vulnerability Scanning view and the advisory-centric view. Installing an advisory remediates all of its CVEs for the affected packages at once.

Edition Requirements

Advisory Management is a Professional+ capability provided by the advisory_engine module. In the Community Edition the advisory API returns HTTP 402 / 403 and the Advisories page and host tab are hidden. Upgrade to a Professional or higher tier to enable it.
