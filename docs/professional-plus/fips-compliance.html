FIPS Compliance Mode Management

Know which hosts are running in FIPS 140 validated-cryptography mode, and turn it on or off from the console. Every agent reports its FIPS posture (this part is free); enabling and disabling FIPS is an Enterprise capability that infers the correct switch mechanism from each host's operating system and dispatches it through SysManage's audited, maintenance-window-aware command queue.

Overview

FIPS 140 is the U.S. federal standard for validated cryptographic modules. Many regulated environments — government, defense, healthcare, finance — require that hosts operate with FIPS mode enabled so that only validated cryptography is used. FIPS Compliance Mode Management gives you a single place to see the FIPS posture of every host and to change it where your license permits.

Per host, SysManage reports one of four states:

  • Enabled — FIPS mode is active (and, on Linux, enforced by the kernel).
  • Available — the host can run FIPS mode (the validated modules are present) but it is currently off.
  • Disabled — the OS supports FIPS but it is not enabled and the modules are not installed.
  • Not applicable — the operating system has no OS-level FIPS mode (macOS and the BSDs).

Detection is built in (and free)

FIPS detection ships in the open-source agent — no license required. On every check-in each agent reports its posture, and the server records it on the host so the dashboard and the per-host tab always reflect reality. Detection uses each platform's authoritative source:

  • Linux (kernel): /proc/sys/crypto/fips_enabled — the definitive "is FIPS enforced right now" flag.
  • RHEL & rebuilds: fips-mode-setup --check.
  • Ubuntu: the Ubuntu Pro fips service status.
  • Windows: the FipsAlgorithmPolicy registry value.
  • macOS & BSD: reported as not applicable.

Fleet dashboard

The FIPS Compliance page in the main navigation summarises the fleet: how many hosts are enabled, available, disabled, or not applicable, followed by a table of every host with its status, the switch mechanism in use, and when it last reported. Use it to answer "which hosts are not in FIPS mode?" at a glance before an audit.

Per-host enable & disable

Open a host's detail page and select the FIPS tab to see its posture — status, mechanism, whether the kernel is enforcing it, and the validated-module version — and, where licensed, to enable or disable FIPS with a single click.

The engine infers the correct mechanism from the host's operating system and runs a pre-flight check (it refuses a no-op or an impossible transition — for example, enabling FIPS on a host that lacks the validated modules) before anything is dispatched:

  • Ubuntu — pro enable fips / pro disable fips
  • RHEL / Fedora / AlmaLinux / Rocky / SUSE — fips-mode-setup --enable / --disable
  • Windows — the FipsAlgorithmPolicy registry value

A reboot is required. Changing FIPS mode only takes full effect after the host restarts, so the console flags the change as reboot-required; the settled posture is reported on the host's next check-in after it reboots.

Scheduled & maintenance-window aware

An enable/disable request is dispatched through SysManage's normal store-and-forward command queue, so it is audited and honours any maintenance window configured for the host — the change is held until the host's next allowed window. Because FIPS changes require a reboot, driving them through change windows keeps the disruption inside your approved maintenance times.

Detection is OSS; enabling is Enterprise

Reporting the FIPS posture of your fleet is part of the free agent, so you always know where you stand. Changing FIPS mode — the enable/disable actions, the fleet FIPS page, and the per-host controls — is an Enterprise capability provided by the compliance_engine module, which performs the OS-to-mechanism inference and pre-flight validation.

Edition Requirements

FIPS detection is available in every edition — the posture is collected and stored for all hosts. FIPS mode management (enable/disable, the FIPS Compliance page, and the host tab's controls) is an Enterprise capability gated behind the FIPS_MODE feature. In lower editions the FIPS management API returns HTTP 402 / 403 and the FIPS Compliance page is hidden. Upgrade to the Enterprise tier to enable it.
